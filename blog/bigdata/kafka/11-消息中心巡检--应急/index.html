10、检查目前正在进行重平衡的消费组 - 需监控

${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server=${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties –describe  --all-groups | grep rebalancing

若显示 “Warning: Consumer group 'XXXX' is rebalancing.”，此通道号在做重平衡，一般几秒内完成。 等两分钟再执行一次，或多次执行，若仍显示此通道号在重平衡，需要联系消息中心项目组及此通道号的应用项目组共同排查。

15、消费组消息堆积阈值检查 (定时任务)

#!/bin/bash

export IP = 本机 kafkaIP:9092 
export KAFKA_HOME="/home/ap/cdakk/kafka/data/kafka_2.12-2.3.2-SNAPSHOT" 
export KAFKA_OPTS="-Djava.security.auth.login.config=${KAFKA_HOME}/config/kafka_client_jaas.conf"

source /etc/profile
source ~/.bash_profile
source /home/ap/cdakk/.kafka_bash_profile

# 获取前一天的日志
previous_date=$(date -d "yesterday" "+% Y-% m-% d")

# 日志重命名
if test -f /home/ap/cdakk/nmv-tools/lag-log/group-lag-$previous_date.log; then
	echo "File exists"
else
	mv /home/ap/cdakk/nmv-tools/lag-log/group-lag.log/home/ap/cdakk/nmv-tools/lag-log/group-lag-$previous_date.log
fi

# 查询消息堆积
${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server=${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties –describe  --all-groups | awk '$1 != "GROUP" && $6 > 100 {print $1, $6}' >> /home/ap/cdakk/nmv-tools/lag-log/group-lag.log

# 删除 30 天前的日志文件
find /home/ap/cdakk/nmv-tools/lag-log/-type f -name "group-lag-*.log" -mtime +30 -exec rm {} +

二、消息中心应急预案

1、如何查看消息中心 SDK 版本

项目组服务中 pom.xml ，mc-java-api，查看版本 version。

2、如何查看消息追踪日志

在 mc-java-api 2.3.2.4 版本开始支持。 新的消息中心 API 添加了消息追踪日志，在进行收发消息时，会在启动目录下生成 trace-producer 和 trace-consumer 两个日志目录。

trace-producer -日期.log 生产者消息追踪日志
trace-consumer -日期.log 消费者消息追踪日志

消息配置： 可以在项目中 resources 目录下添加 tinylog.properties 用来覆盖 mc-java-api 中的日志配置。 具体配置参数参看 svn 查询消息生产消费方法.doc 文档。

3、如何查看生产者的消息发送时间戳

1、项目组使用消息中心的 mc-java-api 版本高于 2.3.2.4 (含) 情况：查看 msg-producer_yyyy-MM-dd_{count}.log 文件中，追踪日志显示的 time 时间值 如何查看消费者的消息消费时间戳 2、项目组使用消息中心的 mc-java-api 版本高于 2.3.2.4 (含) 情况：查看 msg-consumer_yyyy-MM-dd_{count}.log 文件中，追踪日志显示的 time 时间值

4、客户端启动时注册到服务端一直显示 Heartbeat 原因重平衡

在 EDA 客户端的节点挂载一个新的配置文件：mcConsumer.properties，与 mc.properties 同目录 配置文件只有一行内容：

session.timeout.ms=100000

然后重启 EDA 客户端 如何查找 CSID 和子系统名称的对应关系

  • 1、CSID 分为发布 CSID 和订阅 CSID，发布 CSID 格式：T_事件码_发布者传输通道节点号_S，订阅 CSID 格式：T_事件码_订阅者传输通道节点号_R。
  • 2、传输通道号是 8 位：11003201，前 6 位是系统的安全节点号：110032。
  • 3、通过安全节点号查询对应物理子系统：110032 -> N-OTP（经营作战决策）。 PS: eda 统计了母行传输通道节点号与物理子系统以及连接人的文档，方便查询。

12、如何重置某个 Topic 的偏移量为某个指定值或重置为 0

重置到最早: --to-earliest 消费者会从主题的最早消息开始读取，即 CURRENT-OFFSET 0。

${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server ${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties --topic <topic_name> --group <group_name>--reset-offsets --to-earliest --execute

重置到最新： --to-latest 消费者会从主题的最新消息开始读取，即 CURRENT-OFFSET 最新。

${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server ${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties --topic <topic_name> --group <group_name> --reset-offsets --to-latest --execute

指定重置量： --shift-by 【移动量】 即 CURRENT-OFFSET = CURRENT-OFFSET - 移动量

${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server ${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties --topic <topic_name> --group <group_name> --reset-offsets --shift-by <shift_num> --execute

将 Kafka 消费者的偏移量设置到某个时间前： 其中，<kafka_host> 和是 Kafka 服务器的主机名和端口号，<group_name> 是消费者组的名称，<topic_name> 是要消费的主题名称，datetime 是要设置的时间点。请注意，的格式应为 YYYY-MM-DDTHH:mm:ss.sss，例如 2024-01-01T00:00:00.000

${KAFKA_HOME}/bin/kafka-consumer-groups.sh --bootstrap-server ${IP} --command-config ${KAFKA_HOME}/config/client-sasl.properties --topic <topic_name> --group <group_name>--reset-offsets --to-datetime <datetime> --execute

三、EDA 消息消费端优化步骤

蒙商项目中共有 51 个组件使用消息中心客户端进行消息订阅，消息中心项目组提供消费端 SDK，但具体消费逻辑（如消息拉取时间间隔）由各消费端自行实现。为了消除潜在的消费端注册超时风险，需要各订阅方挂载一个优化配置变量 session.timeout.ms。

挂载步骤

1、和项目组沟通，本组件有哪几个服务名称有使用到消息中心 SDK。 2、开阳服务，点开配置管理，查找包含消息中心配置文件 mc.properties 的配置组。 3、点左上角，添加配置文件。 4、填写 mcConsumer.properties 配置文件名称和内容：session.timeout.ms=100000。然后点确认。 5、鼠标点服务管理-服务管理，勾选第一步中有使用消息中心 sdk 的服务名称，点更多，点修改服务。 6、点 containers，点依赖消息中心的服务，下拉点开高级配置，点配置组，点添加。 左边框选择包含 mcConsumer.properties 的配置组，右边框下拉选择 mcConsumer.properties 文件，中间框挂载路径和上面的 mc.properties 一样，可复制下来。 7、如果有多个服务单元使用到消息中心 SDK，有使用消息中心 SDK 的每个服务单元都要按照上面的步骤添加配置文件，并修改服务。 8、重启 EDA 消费端即可。三个部署空间重复操作。若有疑问可联系消息中心项目组。

四、事件管理巡检步骤

4.1、注册中心巡检步骤

1、服务检查

ps -ef |grep ems-regcenter

应有 ems-regcenter 服务进程 2、连通性及域名检查

  • 通过域名访问注册中心（访问成功则注册中心域名生效）。
  • 输入账号密码后能正常登录（与数据库访问正常）。
  • 进入 “传输通道节点” 菜单，修改任意传输通道节点的 “备注” 信息，如增减空格等，修改完成后，在 “事件发布” 菜单中，点击发布，若发布成功，则注册中心与 zk 服务器连接正常，且 zk 服务器域名生效。

4.2、同步集群巡检步骤

4.2.1、服务绿灯检查

执行命令 sh zkServer.sh status，应有如下输出。 Mode:leader 或 follower。

4.2.2、服务健康状态检查

执行命令：telnet 127.0.0.1 2181, 连续多次回车建立连接。 再执行命令：stat。 返回节点信息说明集群正常，按需检查集群其他服务器状态。

4.2.3、服务数据检查

执行命令 sh zkCli.sh。 执行通过后执行 ls /event/app/reg。 页面输出所有传输通道节点号。 执行 get /event/app/reg/{传输通道节点号}。 页面输出指定传输通道节点号的配置内容。

4.2.4、检查存储空间利用率

df -h 各文件系统利用率保持在 80% 以下。
